The deployment keeps timing out at the Bundle step (which is actually the health check after container startup). Build succeeds in seconds, layers push fine, but the app doesn't respond within the 8-minute timeout.

The root cause is likely that the `faster-whisper` / CTranslate2 library is being imported at startup (during Python module loading), which is very slow on cold start. Even though STT is not needed for the health check, Python loads all imports when the app starts.

Please make these changes:

## Fix 1: Lazy-import faster-whisper (MOST IMPORTANT)

In `backend/app/services/whisper_local_service.py`, change it so that `faster-whisper` is NOT imported at module level. Instead, import it only when actually needed (inside the function that does transcription). This prevents the slow CTranslate2 initialization from blocking app startup.

Example pattern:
```python
# DON'T do this at the top of the file:
# from faster_whisper import WhisperModel

# DO this instead — import inside the function:
def transcribe(audio_path: str):
    from faster_whisper import WhisperModel  # lazy import
    # ... rest of the code
Also make sure no other file imports from whisper_local_service at module level in a way that triggers the heavy import chain. Check backend/app/main.py and any router that imports the whisper service.
