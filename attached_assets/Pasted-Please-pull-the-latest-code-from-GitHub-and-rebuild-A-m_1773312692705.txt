What's new — CRAA (Critical Response to Academic Arguments) Practice Mode
A new exercise type craa has been added, following the HKBU UE1 Module 4 speaking test format. Here's what changed:

Backend changes:

database.py — Exercise model has 7 new columns (argument_text, argument_audio_data, argument_audio_type, topic_context, key_claim, preparation_time, response_time). The _run_migrations() function will auto-add these columns to the existing PostgreSQL exercises table on startup.
models/exercise.py — New craa enum value + CRAA Pydantic fields
exercise_router.py — Refactored + new POST/GET /{id}/argument-audio endpoints for uploading/streaming argument audio
practice_router.py — New POST /craa-analyze endpoint that uses CRAA-specific rubric evaluation
llm_service.py — New generate_craa_feedback() function
prompts/craa_feedback.py — New file with CRAA rubric prompt (Summary 40% + Counterargument 30% + Delivery 30%)
Frontend changes:

CRAAPracticePage.tsx — New page with full 4-stage CRAA flow: Intro → Listen → Prepare (countdown) → Record (countdown) → AI Assessment
ExerciseManagePage.tsx — Updated with CRAA configuration form (argument text, audio upload, key claim, timing)
PracticePage.tsx — CRAA exercises auto-redirect to /craa-practice
App.tsx — New /craa-practice route
ExerciseCard.tsx — CRAA icon and label
types/exercise.ts — CRAA type and fields
